================================================================================
23.7 VISUAL PRINCIPLES
================================================================================

1. BREATHING ROOM
```
- Generous padding (16dp minimum)
- Content doesn't feel cramped
- Especially important on small screens
- White space is a feature, not waste
```

2. ORGANISED HIERARCHY
```
- Clear visual structure
- Most important information first
- Progressive disclosure (details on demand)
- Consistent patterns across all screens
```

3. WARM MINIMALISM
```
- Clean, but not sterile
- Organised, but not rigid
- Simple, but not boring
- Functional beauty
```

4. PERFORMANCE-FIRST
```
- No heavy animations on old hardware
- Static icons over animated where possible
- Images lazy-loaded and compressed
- Dark mode saves battery on AMOLED
- Test on 3GB RAM devices
```

5. HONEST STATES
```
- Show real status, not vague assurances
- "Protected" means actually protected
- "47 trackers blocked" not "You're safe!"
- Admit limitations clearly
```

6. ACCESSIBILITY ALWAYS
```
- 4.5:1 contrast minimum
- Touch targets 48dp minimum
- Screen reader labels on all elements
- No information conveyed by color alone
- Scales gracefully to Elder mode
```

23.11 DARK MODE
================================================================================

DARK MODE PALETTE:
```
Background (primary):     #1A1F36 (Circle Deep)
Background (elevated):    #252A40 (cards, modals)
Background (surface):     #2D3349 (inputs, wells)
Text (primary):           #F5F0EB (Circle Warm)
Text (secondary):         #A8ADB8
Dividers:                 rgba(255, 255, 255, 0.1)
```

DARK MODE RULES:
```
1. Don't invert accent colors (Gold stays Gold)
2. Status colors remain the same (semantic meaning)
3. Reduce shadow intensity (nearly invisible)
4. Add subtle borders to cards (1px, 10% white)
5. Test readability on AMOLED screens
```

AUTOMATIC SWITCHING:
```
Options:
- Follow system setting (default)
- Always light
- Always dark
- Scheduled (sunset to sunrise)
```

23.13 VOICE & TONE
================================================================================

BRAND VOICE:
```
CONFIDENT      Not arrogant. We know what we built.
WARM           Human, approachable, not corporate.
HONEST         Admit limitations. No marketing speak.
INCLUSIVE      Everyone belongs in the Circle.
RESPECTFUL     Never condescending. Users are smart.
```

WRITING PRINCIPLES:
```
1. CLEAR OVER CLEVER
   Bad:  "Your digital fortress awaits!"
   Good: "Your phone is protected."

2. ACTIVE VOICE
   Bad:  "Threats have been blocked by the firewall."
   Good: "The Circle blocked 47 threats."

3. SECOND PERSON
   Bad:  "Users can enable protection..."
   Good: "You can enable protection..."

4. NO JARGON (unless necessary)
   Bad:  "C2 beacon intercepted via DNS sinkhole."
   Good: "Blocked a connection to a malware server."
   (Technical details available on "View Details")

5. POSITIVE FRAMING
   Bad:  "Don't let hackers steal your data!"
   Good: "Your data stays with you."
```

ERROR MESSAGES:
```
Bad:  "Error 403: Permission denied"
Good: "This app needs permission to continue.
       [Grant Permission]"

Bad:  "Network request failed"
Good: "Can't connect right now. Check your internet
       and try again. [Try Again]"

Bad:  "Invalid input"
Good: "That doesn't look like an email address.
       Please check and try again."
```

SUCCESS MESSAGES:
```
Bad:  "Operation completed successfully."
Good: "You're protected."

Bad:  "Settings saved."
Good: "Got it. Your changes are saved."

Bad:  "Threat neutralized."
Good: "Threat blocked. You're safe."
```
