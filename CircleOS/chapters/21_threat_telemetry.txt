21.2 WHAT WE COLLECT (OPT-IN)
================================================================================

THREAT INDICATORS (always anonymized):
```java
class ThreatReport {
    // Network indicators
    String destIP;                   // C2 IP address
    String destDomain;               // C2 domain
    int destPort;                    // Port used
    String protocol;                 // HTTP, DNS, binary
    String tlsFingerprint;           // JA3/JA3S hash
    String certHash;                 // SSL cert fingerprint

    // Malware indicators
    String malwareHash;              // SHA256 of APK/binary
    String packageName;              // App package name
    List<String> yaraMatches;        // Which rules matched
    String behaviorSignature;        // Behavioral fingerprint

    // Attack metadata
    String attackType;               // spyware, ransomware, tracker
    String severity;                 // low, medium, high, critical
    long timestamp;                  // Fuzzed to day only
    String countryCode;              // Victim country (2-letter)

    // Traffic samples (sanitized)
    byte[] beaconSample;             // C2 beacon pattern
    byte[] responseSample;           // C2 response pattern
}
```

WHAT WE NEVER COLLECT:
```
âŒ Device identifiers (IMEI, serial, Android ID)
âŒ User accounts or credentials
âŒ Contact lists or names
âŒ Message content
âŒ Photos or files
âŒ Precise location (GPS)
âŒ Browsing history
âŒ Installed apps list
âŒ Usage patterns
âŒ Any PII
```

ANONYMIZATION PIPELINE:
```java
class ThreatAnonymizer {

    ThreatReport anonymize(RawThreatData raw) {
        ThreatReport report = new ThreatReport();

        // Keep threat indicators (the attack)
        report.destIP = raw.destIP;               // Keep full IP
        report.destDomain = raw.destDomain;       // Keep full domain
        report.malwareHash = raw.malwareHash;     // Keep hash

        // Fuzz timestamps
        report.timestamp = fuzzToDay(raw.timestamp);

        // Generalize location
        report.countryCode = raw.countryCode;     // Country only

        // Remove all device identifiers
        // (Not even included in raw data sent)

        // Sanitize traffic samples
        report.beaconSample = sanitizeTraffic(raw.beaconSample);

        return report;
    }

    long fuzzToDay(long timestamp) {
        // Round to start of day (UTC)
        return timestamp - (timestamp % 86400000);
    }

    byte[] sanitizeTraffic(byte[] raw) {
        // Remove any embedded identifiers
        // Keep protocol structure only
        return ProtocolSanitizer.sanitize(raw);
    }
}
```

21.6 CANARY TOKEN SYSTEM
================================================================================

Canary tokens are unique identifiers placed in honeypot data that "phone home"
when accessed, revealing that data was exfiltrated and who accessed it.

CANARY TYPES:
```java
enum CanaryType {
    EMAIL,       // Unique email address
    PHONE,       // Unique phone number
    DOCUMENT,    // PDF/doc with web beacon
    URL,         // Unique URL that logs access
    DNS          // Unique hostname that logs resolution
}

class CanaryToken {
    String id;
    CanaryType type;
    String value;              // The actual canary
    String deviceHash;         // Hashed device ID (for correlation)
    long created;
    boolean triggered;
    TriggerInfo lastTrigger;
}

class TriggerInfo {
    long timestamp;
    String sourceIP;           // Who accessed it
    String userAgent;          // Browser/app
    String referrer;           // How they got there
    Map<String, String> headers;
}
```

CANARY GENERATION:
```java
class CanaryGenerator {

    // Email canary
    CanaryToken generateEmailCanary() {
        String unique = generateUniqueId();  // e.g., "x7k2m9"
        String email = unique + "@canary.circelos.org";

        // Register with mail server to track delivery attempts
        registerEmailCanary(email, deviceHash);

        return new CanaryToken(EMAIL, email);
    }

    // Document canary (PDF with hidden beacon)
    CanaryToken generateDocumentCanary(String filename) {
        String trackingId = generateUniqueId();
        String beaconUrl = "https://canary.circelos.org/t/" + trackingId + "/o.gif";

        // Create PDF with hidden image beacon
        byte[] pdf = PdfGenerator.createWithBeacon(filename, beaconUrl);

        return new CanaryToken(DOCUMENT, filename, pdf, trackingId);
    }

    // DNS canary (unique subdomain)
    CanaryToken generateDnsCanary() {
        String unique = generateUniqueId();
        String hostname = unique + ".dns.canary.circelos.org";

        // DNS server logs all resolution attempts

        return new CanaryToken(DNS, hostname);
    }
}
```

CANARY TRACKING SERVER:
```
Located at: canary.circelos.org

When canary triggered:
1. Log: timestamp, source IP, headers
2. Lookup: which device created this canary
3. Alert: Send push notification to device
4. Store: Evidence for user review
```

21.8 INTEGRATION WITH FIREWALL
================================================================================

AUTOMATIC BLOCKING:
```java
class ThreatIntelIntegration {

    // Called by CircleFirewallService for every connection
    ThreatMatch checkConnection(String domain, String ip) {
        // Check domain
        ThreatDomain td = threatDb.getDomain(domain);
        if (td != null) {
            return new ThreatMatch(td.threatType, td.severity, td.campaign);
        }

        // Check IP
        ThreatIP ti = threatDb.getIP(ip);
        if (ti != null) {
            return new ThreatMatch(ti.threatType, ti.severity, ti.campaign);
        }

        return ThreatMatch.NONE;
    }

    // Called when APK is installed
    ThreatMatch checkApk(String sha256) {
        ThreatHash th = threatDb.getHash(sha256);
        if (th != null) {
            return new ThreatMatch(th.threatType, th.severity, th.family);
        }
        return ThreatMatch.NONE;
    }
}
```

FIREWALL ACTIONS:
```
CRITICAL threat match:
  â†’ Block immediately
  â†’ Alert user
  â†’ Offer to send to Malware Jail

HIGH threat match:
  â†’ Block
  â†’ Silent notification
  â†’ Log

MEDIUM threat match:
  â†’ Send to Traffic Lobby
  â†’ User decides

LOW threat match:
  â†’ Allow (trackers, etc.)
  â†’ Log for Privacy Dashboard
```

21.9 PRIVACY GUARANTEES
================================================================================

TECHNICAL GUARANTEES:
```
1. NO DEVICE FINGERPRINTING
   - No unique identifiers sent
   - No hardware IDs
   - No persistent cookies

2. NO CORRELATION
   - Each report independent
   - No session linking
   - IP not logged server-side

3. NO CONTENT
   - Traffic samples sanitized
   - No user data in reports
   - Protocol structure only

4. DIFFERENTIAL PRIVACY
   - Add noise to statistics
   - k-anonymity for queries
   - Cannot identify individuals

5. TRANSPARENCY
   - Open source client code
   - Published API spec
   - Regular audits
```

LEGAL FRAMEWORK:
```
- POPIA compliant (South Africa)
- GDPR compliant (for EU users)
- No data sold to third parties
- No government data sharing (without warrant)
- Annual transparency report
- User can request data deletion
```
